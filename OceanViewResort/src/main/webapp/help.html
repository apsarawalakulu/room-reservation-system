Digital Subsystem Navigation Reference

1. Welcome to the New Digital Portal

This system replaces the paper ledgers. It guarantees atomic isolation of room allocations preventing double-bookings instantly while maintaining a secure logging stream across checkouts and automatic receipt compilations.

2. Glossary of Terminology

  • Reservation Code (RES-#####): Primary relational tracking string identifying a specific guest's visit timeframe linked accurately to system modules.
  • Standard Single/Double Rooming: Baseline hotel allocations tied directly to max constraints preventing single rooms seating 4 guests.
  • Deluxe/Suite Rooming: Larger formats enabling scaled occupancy thresholds commanding variable rate layers.
  • LKR Currency: All generated computational logic prints in Sri Lankan Rupees.
  • Check-in/Check-out Scope: Strictly bound calendar events forcing mathematical differences yielding correct Night calculations for taxes.

3. Initiating a Booking Command

Triggering the New Reservation button brings a strict sequential form. Supply Guest constraints accurately without violating formats (ie: Phones must naturally be numeric). Selecting check bound dates opens the Inventory fetch stream where pressing Find Available Rooms scans DB overlaps validating what allocations strictly exist before allowing submission.

4. Monitoring Operations Ledger

The All Reservations index acts as a global lookup filtering actively running guests vs historical footprints (Cancelled/Checked Out) ensuring accurate visual rendering across thousands of rows quickly using paginated network limits.

5. Finalization and Tax Exports

Upon review screens, Cancel Booking forcibly halts the sequence destroying matching reservations while extracting formal reason traces. Contrastly, Generate Bill abstracts core variables running VAT scaling logic formatting outputs correctly hiding global UI arrays strictly maintaining printable documents mapped to paper constraints perfectly.
